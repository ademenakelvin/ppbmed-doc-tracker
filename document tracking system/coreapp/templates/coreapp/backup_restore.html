Recovery guide

Backup and restore checklist

1. Create backups regularly. Run `backup-system.ps1` and copy the resulting folder off the server.
2. Verify both database and media are present. A database without attachments, or attachments without the database, is not a full recovery point.
3. Restore carefully. Stop the app first, replace `db.sqlite3` and the `media` folder from the backup set, then start the app and test key pages.
4. Keep more than one backup. Do not rely on only the latest folder in case the last backup was taken after a mistake.
